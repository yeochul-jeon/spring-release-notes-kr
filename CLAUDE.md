# Spring Release Notes 프로젝트 가이드

## 프로젝트 개요
Spring Framework(4.x~7.0)와 Spring Boot(2.x~4.0)의 버전별 릴리즈 노트를 한국어로 정리하는 문서 프로젝트.

## 문서 작성 규칙

### 언어
- 모든 설명은 **한국어**로 작성
- 코드 예제의 주석도 한국어
- 전문 용어는 처음 등장 시 간단히 설명

### 대상 독자
- 초보 개발자 — 쉽고 상세한 설명 지향

### 문서 구조
모든 버전 문서는 아래 템플릿을 따름:
1. 개요 (2~3문장 요약)
2. 주요 변경사항 (코드 예제 포함)
3. 새로운 기능
4. 개선사항
5. 폐기(Deprecated) 및 제거(Removed) 항목
6. 호환성
7. 참고 자료

### Mermaid 다이어그램
아래 경우에 적극 활용:
- 아키텍처 변경
- 버전 간 의존성/호환성 관계
- 새로운 처리 흐름
- 모듈 구조 변경

### 길이 관리
- 단일 버전 문서가 300줄 초과 시 주제별 하위 문서로 분리

### 정보 출처
- 공식 GitHub Wiki 릴리즈 노트
- Spring 공식 블로그
- 추측 금지 — 확인된 사실만 기재

## 디렉토리 구조
```
spring-release-notes/
├── spring-framework/          # Spring Framework 릴리즈 노트
│   ├── 4.x/ ~ 7.x/
├── spring-boot/               # Spring Boot 릴리즈 노트
│   ├── 2.x/ ~ 4.x/
└── migration-guides/          # 메이저 버전 간 마이그레이션 가이드
```

## 링크 규칙
- 문서 간 참조는 **상대 경로** 사용
- 예: `[Spring Boot 3.0](../spring-boot/3.x/3.0.md)`
